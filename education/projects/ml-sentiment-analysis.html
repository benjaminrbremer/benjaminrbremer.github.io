Sentiment Analysis on Financial News Using BERT

Fine-tuning a pre-trained BERT model to classify market-moving news headlines as bullish or bearish — and exploring where transformer-based NLP outperforms classical bag-of-words approaches.

Overview

This project was completed as the final assignment for my Machine Learning course. The goal was to build a text classification pipeline capable of determining whether a financial news headline carries a positive (bullish) or negative (bearish) signal for the market. The motivation was practical: sentiment extracted from news has been widely studied as a predictor of short-term equity price movement.

I fine-tuned bert-base-uncased from Hugging Face Transformers on a labeled dataset of approximately 4,800 financial news headlines. The final model achieved 87.2% accuracy on the held-out test set, compared to 64.1% for a TF-IDF + logistic regression baseline — a 23-point improvement.

Problem Statement

Financial news headlines are short, domain-specific, and often rely on implicit cues that classical bag-of-words models struggle to capture. Phrases like "misses estimates" and "beats expectations" both contain the word "estimates," but carry opposite sentiment. A model that understands context — not just word frequency — should handle these cases better.

The task was framed as a binary classification problem: given a headline, predict bullish (1) or bearish (0). A neutral class was excluded to simplify the task and because unlabeled examples in the source dataset were ambiguous.

Data

The dataset was sourced from a combination of the Financial PhraseBank dataset (Malo et al., 2014) and headlines scraped from a financial news aggregator between 2019–2023. After removing neutral-labeled samples and deduplication, the final dataset contained:

  • Train: 3,840 examples (80%)
  • Validation: 480 examples (10%)
  • Test: 480 examples (10%)

The class distribution was roughly balanced at 52% bullish / 48% bearish. No oversampling was required.

Approach

I used bert-base-uncased as the backbone — 110M parameters, trained on BooksCorpus and English Wikipedia. A linear classification head was added on top of the [CLS] token representation and trained end-to-end using cross-entropy loss.

Training Setup

  • Optimizer: AdamW with weight decay 0.01
  • Learning rate: 2e-5 with linear warmup over 10% of steps
  • Batch size: 16 (gradient accumulation × 2 for effective batch of 32)
  • Epochs: 4 (early stopping on validation loss with patience 2)
  • Hardware: Single NVIDIA A100 40GB (university HPC cluster)
from transformers import BertForSequenceClassification, AdamW
from torch.utils.data import DataLoader

    "bert-base-uncased", num_labels=2

Results

  • BERT accuracy: 87.2% vs. baseline 64.1%
  • BERT F1 (macro): 0.871 vs. baseline 0.638
  • BERT AUC-ROC: 0.934 vs. baseline 0.701

Error analysis revealed that the model struggled most on headlines with sarcasm or implicit comparisons. These cases likely require broader document context or domain-specific pre-training.

Takeaways

  • Transfer learning from large pre-trained models dramatically reduces the data requirements for downstream classification tasks. 4,800 examples produced a strong model.
  • Learning rate and warmup scheduling matter significantly for BERT fine-tuning. Higher learning rates caused catastrophic forgetting of pre-trained representations.
  • The gap between validation accuracy and test accuracy was small (≈0.5%), suggesting no significant data leakage or distribution shift.

If I revisited this project, I'd explore FinBERT — a BERT variant pre-trained on financial text — and experiment with multi-class sentiment using a larger, more balanced dataset.
